NAT Types

NAT (Network Address Translation) type affects whether a Comrex device can establish and receive codec connections. More restrictive NAT types may require STUN/TURN traversal assistance via Switchboard.

Open Open Internet
No NAT in use. The device has a public IP and accepts inbound connections directly. Best possible connectivity.
Full Cone Full Cone NAT
Any external host can send packets to the device's mapped port. Excellent for codec connections — reliable peer-to-peer without traversal assistance.
Restr. Cone Restricted Cone NAT
Only hosts the device has previously contacted can send inbound packets. Generally works well for Comrex connections initiated through Switchboard.
Port Restr. NAT Port Restricted Cone NAT
Like Restricted Cone, but the source port must also match. Most consumer and business routers use this type. Switchboard traversal assistance is usually sufficient.
Symmetric Symmetric NAT
Each outbound connection gets a unique external port mapping. Hardest type for peer-to-peer — TURN relay via Switchboard is typically required for reliable connections.
